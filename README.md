## 2. 核心 Prompt 迭代记录
### 初代 Prompt
帮我写一个Python的学生信息与考场管理系统的ExamSystem类的文件读取方法，读取指定的学生名单txt文件，把数据存起来，要处理文件找不到的异常。

### AI 生成的问题/缺陷
1. 未考虑学生名单文件的**制表符分隔**格式，使用`split()`按空格切割，导致字段解析错位；
2. 未跳过文件第一行的表头，将表头的文字内容当作学生数据处理，后续转换整型时触发报错；
3. 仅捕获了`FileNotFoundError`，未考虑数据格式转换的`ValueError`，异常处理不全面；
4. 直接将解析后的数据存为列表，未按要求转换为已定义的`Student`类对象，违反OOP编程规范。

### 优化后的 Prompt (追问)
帮我优化ExamSystem类的文件读取方法，要求：1. 按**制表符\t**切割文件内容，跳过第一行的表头；2. 将解析后的每一条学生数据转换为已定义的Student类对象，存入实例列表；3. 除了捕获FileNotFoundError，还要捕获整型转换的ValueError并给出友好提示；4. 仅使用Python标准库，文件读写指定UTF-8编码防止中文乱码；5. 解析的字段顺序严格对应“序号、姓名、性别、班级、学号、学院”，班级字段转为整型，学号按字符串存储。

## 3. Debug 与异常处理记录
### 报错类型/漏洞现象
运行AI生成的文件读取代码时，触发`ValueError: invalid literal for int() with base 10: '性别'`报错，程序终止运行，无法读取任何学生数据。

### 解决过程
首先我通过查看Python的Traceback报错信息，定位到报错行是将解析后的字段转换为班级整型的代码行，结合学生名单文件内容，发现AI的代码**未跳过表头行**，把表头中的“性别”字符串当作班级数字进行整型转换，因此触发报错；同时还发现代码使用`split()`按空格切割，而文件实际是制表符分隔，会导致字段顺序完全混乱。
我先手动修改了代码，在遍历文件行时通过`enumerate`获取行号，让行号为0时直接跳过表头；再将`split()`修改为`split("\t")`按制表符切割；最后按文件真实的字段顺序取值，将“班级”对应的字段转为整型，其他字段按对应类型存储。修改后重新运行代码，成功解决该报错，能正常读取所有10名学生的信息并转换为`Student`对象。

## 4. 人工代码审查 (Code Review)
以下为AI生成的**随机打乱学生顺序+生成考场安排表**核心逻辑代码，我已添加逐行中文注释，完整理解其运行机制：
```python
def generate_exam_table(self):
    # 前置校验：判断学生列表是否为空，无数据则直接返回False，不执行后续操作
    if not self.student_list:
        print("❌ 暂无学生数据，无法生成考场安排表！")
        return False
    # 使用random.sample无放回抽样，打乱学生列表顺序，结果存入实例属性供准考证复用
    # 第一个参数为原列表，第二个参数为抽取数量，等于原列表长度时即实现完全打乱
    self.shuffled_students = random.sample(self.student_list, len(self.student_list))
    # 获取当前系统时间，按作业要求的格式"YYYY-MM-DD HH:MM:SS"格式化
    create_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    # 调用类的静态方法拼接文件路径，根目录下生成“考场安排表.txt”，跨系统兼容
    table_path = self.join_path(".", "考场安排表.txt")

    try:
        # 以写入模式打开文件，指定UTF-8编码防止中文姓名/学院乱码
        with open(table_path, "w", encoding="utf-8") as f:
            # 按要求在文件第一行写入格式化的生成时间
            f.write(f"生成时间：{create_time}\n")
            # 遍历打乱后的学生列表，enumerate从1开始计数，作为考场座位号
            for seat_num, stu in enumerate(self.shuffled_students, 1):
                # 按“座位号 姓名 学号”格式写入，制表符分隔让排版整洁
                f.write(f"{seat_num}\t{stu.name}\t{stu.student_id}\n")
        # 生成成功后打印提示信息和文件绝对路径，方便用户查找
        print(f"\n✅ 考场安排表生成成功！")
        print(f"📁 文件路径：{os.path.abspath(table_path)}")
        return True # 返回True标识，供准考证方法判断是否生成成功
    # 单独捕获文件写入权限不足的异常，给出精准的错误提示
    except PermissionError:
        print(f"❌ 生成失败：无文件写入权限，请检查目录权限！")
        return False
    # 捕获其他未知的文件写入异常，兜底保证程序不崩溃
    except Exception as e:
        print(f"❌ 考场安排表生成失败：{e}")
        return False
```
